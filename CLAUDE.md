## Project

Godot 4.6 game — a Doodle Jump-style vertical scroller named `dragon_test`. Uses the **GL Compatibility** renderer and **Jolt Physics**. Design resolution: **360 × 640 (9:16)** with stretch mode `canvas_items / expand` for mobile.

## Running the project

- Open in Godot 4.6 editor: `godot --path /Users/chlienoi-imac/Desktop/Godot/Game-01/dragon-test`
- Run: **F5** (project) or **F6** (current scene)
- There are no build steps or test commands — GDScript errors surface in the editor's Output/Debugger panel.

## Architecture

```
main.tscn  (Node2D + main.gd)
├── Camera2D          — repositioned each frame by main.gd
├── Player            — instance of player.tscn
└── Platforms         — Node2D container; platforms added/removed dynamically
```

**main.gd** owns the game loop:
- `_ready()` seeds the first batch of platforms from the player's start position upward.
- `_process()` scrolls the camera up (one-way, tracks `player.position.y + vp_h * 0.25`), generates new platforms ahead of the camera, culls platforms that scroll off the bottom, and reloads the scene on game-over.
- `_spawn_platform(y)` instantiates `platform.tscn` at a random X within `PLATFORM_MARGIN` insets.

**player.gd** (CharacterBody2D):
- Gravity + `move_and_slide()` each physics frame.
- Auto-jumps with `JUMP_VELOCITY` whenever `is_on_floor()` is true — no explicit jump input needed.
- Input: `Input.get_accelerometer()` on mobile; `ui_left` / `ui_right` actions on desktop.
- Wraps horizontally when the player exits the left/right edge.
- Visual drawn via `_draw()` (no sprite asset required).

**platform.gd** (StaticBody2D):
- Stateless — only implements `_draw()` for the green rectangle visual.
- The `CollisionShape2D` child has `one_way_collision = true` so the player can jump through from below but lands on top.

## Key tuning constants

| File | Constant | Value | Effect |
|------|----------|-------|--------|
| player.gd | `JUMP_VELOCITY` | -800 | Jump height ≈ 178 px |
| player.gd | `GRAVITY` | 1800 | Snappy fall |
| player.gd | `MOVE_SPEED` | 300 | Horizontal speed |
| main.gd | `SPACING_MIN/MAX` | 100–150 | Vertical gap between platforms |
| main.gd | `PLATFORM_MARGIN` | 65 | Min distance from screen edge |

## GDScript conventions

- Static typing throughout (`-> void`, `: float`, `:=` inference).
- `_ready()` for one-time init, `_physics_process(delta)` for physics, `_process(delta)` for non-physics per-frame logic.
- `.uid` sidecar files are auto-generated by the editor; commit them alongside their `.gd` file so UIDs stay stable across machines.
- `.godot/` is git-ignored — never commit it.
